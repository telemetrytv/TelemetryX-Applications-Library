Overview

The TelemetryX Development Harness is a sophisticated browser-based development environment that simulates the TelemetryX platform for local application development. It provides a complete mock SDK environment, hot reload capabilities, and dual-iframe layout for simultaneous render and settings view development.

Iframe Configuration

Render Iframe 75% width

  • Purpose: Main application display
  • Aspect Ratio: 16:9 (1920x1080)
  • URL Pattern: /apps/{appId}/index.html
  • Sandbox: Scripts, same-origin, forms, popups, modals

Settings Iframe 25% width

  • Purpose: Configuration interface
  • Height: Full container height
  • URL Pattern: /apps/{appId}/settings.html
  • Sandbox: Same as render iframe

Build System Integration

The harness integrates with each application's build system, allowing you to build applications directly from the browser interface.

Build Process

  1. Click the "Build" button in the harness header
  2. Server executes npm run build in the application directory
  3. Build output streams to the console in real-time
  4. On success, the application automatically reloads
  5. Built files are served from applications/{app}/dist/

Build Configuration

Each application must have:

  • A build script in package.json
  • Output directory set to dist/
  • Multi-entry points for render and settings

Troubleshooting

Common Issues and Solutions

Port Conflicts

Problem: Port 3000 or 3001 already in use

Solution: Check for other processes and kill them, or modify the port in server.js

Hot Reload Not Working

Problem: Changes not triggering browser refresh

Solution: Check WebSocket connection in browser console, ensure port 3001 is accessible

Application Not Loading

Problem: Dropdown shows no applications

Solution: Ensure applications are built (npm run build in app directory)

SDK Errors

Problem: SDK methods returning undefined

Solution: Check that bridge-stub.js is loaded and Bridge connection is established

Build Failures

Problem: Build button shows errors

Solution: Check application's package.json for proper build script and dependencies
